创始成员

四位创始成员，四种力量。

ADPS 不是一个人的个人英雄叙事，而是一个工程共同体的出海叙事。 四位创始成员代表四种力量：架构语言、软件工程、开源生态、企业落地。

黄佳

框架 · 写作 · 全球叙事接口

新加坡 A*STAR AI 研究员，Manning 《Designing AI Agents》 作者，《Agent 设计模式》作者，极客时间《Claude Code 工程化实战》课程作者。

在 ADPS 中的角色：框架提出者、模式语言整理者、全球叙事接口。

关键贡献：

  • 提出并公开双轴框架：认知功能 × 执行拓扑。
  • 将 28 个 Agent 设计模式组织成可讨论、可评审、可迁移的架构语言。
  • 通过中文书、极客时间课程、企业培训、Manning MEAP、arXiv 论文和个人网站，把中国工程师实践推向全球读者。

茹炳晟

软件工程 · 研发效能 · 真实项目

腾讯 Tech Lead，腾讯研究院特约研究员，腾讯集团技术委员会委员，复旦大学 CodeWisdom 团队成员， 国内软件工程、研发效能、AI 赋能研发领域的重要实践者和传播者。

公开资料显示，他还担任中国计算机学会 TF 研发效能 SIG 主席、腾讯云架构师技术同盟入会成长主席、 中国通信标准化协会 TC608 云计算标准和开源推进委员会云上软件工程工作组副组长， 并长期作为技术峰会出品人、联席主席、Keynote 演讲嘉宾与技术图书作者/译者活跃在社区中。

在 ADPS 中的角色：软件工程方法论锚点、研发效能与复杂系统评估视角。

关键贡献：

  • 把 Agent 放回真实软件研发现场，而不是停留在玩具题和 demo。
  • 从研发效能、软件复杂性、组织协作和工程评测角度，为 Agent 设计模式提供现实边界。
  • 连接 CodeWisdom、腾讯和中国软件工程社区，让 ADPS 不只是 AI 圈的概念，而是软件工程共同体的延伸。

姜宁 / Ning (Willem) Jiang

开源 · Harness · 工作流

Apache 软件基金会长期贡献者，Apache Camel 社区重要成员，开源社区活跃实践者； 公开资料显示，他参与 DeerFlow 2.0 相关公开活动，并在社交资料中标注 DeerFlow developer。

在 ADPS 中的角色：开源 Harness 实践者、多 Agent 与工作流工程连接者。

关键贡献：

  • 把 ADPS 的模式语言连接到可运行、可部署、可社区协作的 Harness 工程。
  • 从 Apache Camel 到 CAMEL-AI/DeerFlow 这类 Agent 生态，提供跨代工程经验：路由、编排、集成、工作流、社区治理。
  • 让"Agent 设计模式"不只停留在书和演讲，而是能进入开源项目、开发者社区和真实工具链。

梁博

企业 SaaS · FinTech · 生产反馈

上海东方屹腾科技有限公司副总裁，创办期股东与合伙人，负责技术和产品。东方屹腾核心团队来自银行 互联网渠道领域，聚焦银行泛金融生态场景 SaaS 服务；其薪享通 SaaS 服务矩阵已与 20+ 银行 建立联合运营体系，服务 5 万+ 企业客户，并正在把 SaaS 产品矩阵全面转型为 Agents 体系。

在 ADPS 中的角色：企业级落地反馈、金融 SaaS 产品转型、真实业务约束来源。

关键贡献：

  • 把 ADPS 的模式语言放进银行泛金融、HR SaaS、薪酬、考勤、个税、薪资代发等复杂业务流程里验证。
  • 提供从 SaaS 到 Agents 的转型场景：不是"加一个聊天入口"，而是把企业服务流程改造成可预算、可观测、可授权、可复盘的智能系统。
  • 作为实战型产品与技术伙伴，持续把真实企业问题、架构反馈和产品边界带回框架。

四种力量合在一起

四位创始成员合在一起，代表四种力量：

  1. 架构语言——把混乱经验沉淀成双轴框架、模式矩阵和设计哲学。
  2. 软件工程——把 Agent 放回研发效能、复杂系统和真实项目评估中。
  3. 开源生态——把模式语言和可运行 Harness、社区协作、工程实现连接起来。
  4. 企业落地——把银行、泛金融、SaaS 和组织转型中的真实约束反哺框架。
