Agent Design Patterns Society

沉淀通用范式，做可成长的 Agent 系统。

ADPS 是一个小而认真的研究共同体。我们把中国工程师的一线 agent 实践、软件工程方法、开源 Harness 经验和 企业级落地反馈，沉淀为全球工程师都可以使用、可以讨论、可以评审、可以传承的架构语言。

我们不急着给自己封号。我们只做一件事：把 agent 时代反复出现的架构问题，整理成 工程师能复用、能争论、能评审、能传承的模式语言。

我们做什么

ADPS 把反复出现的 agent 架构问题整理成一套连贯的模式语言。工作围绕三件交付物展开：

  • 模式目录——28 个命名模式落在 7×6 矩阵 （认知功能 × 执行拓扑）上，每一个都有可运行的实现，以及来自真实生产代码的工程切片。
  • 模式选型方法论——6 步流程加 5 条定律， 把"我要做一个 X 的 agent"在十五分钟内变成候选模式清单。
  • 生产级 Harness——开源参考实现， 展示模式在成本、时延、治理约束下如何组合。

框架

双轴框架——认知功能 × 执行拓扑——首次发表在论文 A Two-Dimensional Framework for AI Agent Design Patterns（Huang & Zhou， arXiv:2605.13850）， 并在 Manning 的英文书 Designing AI Agents 中进一步展开。ADPS 现在作为共同体资产持续维护这套目录。
