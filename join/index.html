Join ADPS

How to participate in the work.

ADPS is open to engineers and organizations with field experience building production-grade agent systems — particularly at scale, in regulated industries, or under hard reliability constraints. The community is small by design. Membership is by contribution, not application.

Three ways to contribute

1. Pattern contributions on GitHub

The pattern catalog lives at huangjia2019/agent-design-patterns. Open issues to propose new patterns, suggest renames, or add engineering slices from real production codebases. PRs welcome.

2. Case studies and post-mortems

If you have a production agent system that runs into a pattern problem (or solves one), we want to read about it. Send a short write-up — what topology, what cognitive functions, what broke, what worked, what the cost shape looks like.

3. Harness implementations

Open-source harness work — the runtime layer that turns a pattern into something deployable — is the moat. If you maintain or contribute to an agent runtime, framework, or workflow engine, we are interested in mapping its capabilities onto the matrix.

Get in touch

Until the ADPS organization domain is fully set up, reach out via:

  • GitHub Issues on agent-design-patterns
  • Substack: agentpatterns.substack.com
  • Or contact any of the founders directly.
